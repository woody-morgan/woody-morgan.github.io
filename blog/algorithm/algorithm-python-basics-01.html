Python Basics - Input(ADV)

word = input().upper()
unique_word = list(set(word))

cnt_list = []

for c in unique_word:
    cnt = word.count(c)
    cnt_list.append(cnt)


if cnt_list.count(max(cnt_list)) > 1:
    print('?')
else:
    max_index = cnt_list.index(max(cnt_list))
    print(unique_word[max_index])

-> remove duplicated item using set
-> count max item using count => if it is more than 1, print '?'
-> else, find the index of max item in set list and print it

Same Result but different(Pythonic)

like_other_lang.py
def get_reverse(n):
    ret = 0
    while n > 0:
        remain = n % 10
        ret *= 10
        ret += remain
        n = n // 10
    return ret


n, m = map(int, input().split())

n = get_reverse(n)
m = get_reverse(m)

if n > m:
    print(n)
else:
    print(m)
pythonic.py
a, b = input().split()

a = a[::-1]
b = b[::-1]

print(max(a, b))

Python3 vs Pypy3

Python3 and Pypy3 have different performance. Pypy3 is faster than Python3 when the python code has many loops, more space to cache. It is gained by JIT compiler, but it uses more memory than Python3.

sorted.py
from sys import stdin, stdout

n = int(input())

nums = []
for i in range(n):
    nums.append(int(stdin.readline()))

for n in sorted(nums):
    stdout.write(str(n)+"\n")

Pypy3 is 1.7 times faster than Python3 with the same code.

Tricky Python3

1181.py
from sys import stdin
n = int(input())

w_list = []

for i in range(n):
    w_list.append(stdin.readline().rstrip())

w_list = list(set(w_list))

w_list.sort(key=lambda word: (len(word), word))

for i in w_list:
    print(i)

I thought this code is faster than calling sort function twice. Because it is using lambda & doesn't need to iterate the list twice.

1181.py
from sys import stdin
n = int(input())

w_list = []

for i in range(n):
    w_list.append(stdin.readline().rstrip())

w_list = list(set(w_list))
w_list.sort()
w_list.sort(key=len)

for i in w_list:
    print(i)
